36 /// Replace the `loop` with `newIterOperands` added as new initialization
37 /// values. `newYieldValuesFn` is a callback that can be used to specify
38 /// the additional values to be yielded by the loop. The number of
39 /// values returned by the callback should match the number of new
40 /// initialization values. This function
41 /// - Moves (i.e. doesnt clone) operations from the `loop` to the newly created
42 /// loop
43 /// - Replaces the uses of `loop` with the new loop.
44 /// - `loop` isnt erased, but is left in a "no-op" state where the body of the
45 /// loop just yields the basic block arguments that correspond to the
46 /// initialization values of a loop. The loop is dead after this method.
47 /// - If `replaceIterOperandsUsesInLoop` is true, all uses of the
48 /// `newIterOperands` within the generated new loop are replaced
49 /// with the corresponding `BlockArgument` in the loop body.

51  OpBuilder &b, Location loc, ArrayRef<BlockArgument> newBBArgs)>;

57 /// Update a perfectly nested loop nest to yield new values from the innermost
58 /// loop and propagating it up through the loop nest. This function
59 /// - Expects `loopNest` to be a perfectly nested loop with outer most loop
60 /// first and innermost loop last.
61 /// - `newIterOperands` are the initialization values to be used for the
62 /// outermost loop
63 /// - `newYielValueFn` is the callback that generates the new values to be
64 /// yielded from within the innermost loop.
65 /// - The original loops are not erased, but are left in a "no-op" state where
66 /// the body of the loop just yields the basic block arguments that correspond
67 /// to the initialization values of a loop. The original loops are dead after
68 /// this method.
