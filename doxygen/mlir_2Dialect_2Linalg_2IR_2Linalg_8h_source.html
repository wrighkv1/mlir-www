34 /// Returns the name mangled library call name to disambiguate between different
35 /// overloads at the C level. The name mangling scheme is basic and uses MLIR
36 /// type names:
37 /// 1. form a string which is the concatenation of the linalg op name with all
38 /// the operand type names, separate by underscores;
39 /// 2. drop the `linalg.` prefix, and the `<`, `>`, `?` symbols from the type.
40 /// Assumes `op` is a LinalgOp.
41 ///
42 /// Examples:
43 ///
44 /// 1. linalg.fill(%f, %A) : f32, memref<f32>
45 /// name mangles into `linalg_fill_f32_viewf32`
46 ///
47 /// 2. linalg.dot %A, %B, %C :
48 /// (memref<?xf32, stride_specification>,
49 /// memref<?xf32, stride_specification>, memref<f32>)
50 /// name mangles into `linalg_dot_viewxf32_viewxf32_viewf32`
51 ///
52 /// 3. linalg.matmul(...) :
53 /// memref<?x?xf32, stride_specification>,
54 /// memref<?x?xf32, stride_specification>,
55 /// memref<?x?xf32, stride_specification>
56 /// name mangles into `linalg_matmul_viewxxf32_viewxxf32_viewxxf32`
57 std::string generateLibraryCallName(Operation *op);
